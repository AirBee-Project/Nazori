# 開発規則

このリポジトリでは、変換系の実装を次の方針で整理する。

- `formats` の中にデータ変換を書く。
- 最終的に公開する関数は、変換ソースがわかるシンプルな名前にする。PLATEAU なら `plateau_bldg` のようにする。
- 様々な環境で実行できるように、入力はファイルではなく文字列やバイト列を受け取る。
- 単一ファイルで複雑になる場合はフォルダにしてよいが、表層関数は `mod.rs` に置く。
- 関数やファイルの粒度は、コードが読みやすい大きさに保つ。
- `formats` の中に共通依存を増やしすぎない。各変換関数はできるだけ独立させる。
- 汎用的な関数が必要な場合は `utils` フォルダに置く。
